策略迭代算法（Policy Iteration Algorithm）

在强化学习中，除了值迭代算法之外，策略迭代算法同样是求解最优策略和最优状态价值的经典方法。策略迭代的核心思想是交替进行策略评估（policy evaluation）和策略改进（policy improvement），不断提升策略的表现，最终收敛到最优策略。

策略迭代算法的起点是一个随机初始化的策略 π0\pi_0π0​。在每一次迭代中，首先进行策略评估，即在当前策略 πk\pi_kπk​ 下，求解其对应的状态价值函数 vπkv_{\pi_k}vπk​​。这一步实际上就是求解如下贝尔曼方程：

vπk=rπk+γPπkvπkv_{\pi_k} = r_{\pi_k} + \gamma P_{\pi_k} v_{\pi_k} vπk​​=rπk​​+γPπk​​vπk​​

对于这个方程，我们可以采用闭式解：

vπk=(I−γPπk)−1rπkv_{\pi_k} = (I - \gamma P_{\pi_k})^{-1} r_{\pi_k} vπk​​=(I−γPπk​​)−1rπk​​

也可以采用迭代法：

vπk(j+1)=rπk+γPπkvπk(j),j=0,1,2,…v_{\pi_k}^{(j+1)} = r_{\pi_k} + \gamma P_{\pi_k} v_{\pi_k}^{(j)}, \quad j = 0, 1, 2, \ldots vπk​(j+1)​=rπk​​+γPπk​​vπk​(j)​,j=0,1,2,…

当 vπk(j)v_{\pi_k}^{(j)}vπk​(j)​ 收敛时，即可得到当前策略的状态价值。

完成策略评估后，进入策略改进阶段。此时我们利用刚刚得到的 vπkv_{\pi_k}vπk​​，在每个状态下选择能够最大化长期回报的动作，从而得到新的策略 πk+1\pi_{k+1}πk+1​：

πk+1=arg⁡max⁡π(rπ+γPπvπk)\pi_{k+1} = \arg\max_{\pi} (r_\pi + \gamma P_\pi v_{\pi_k}) πk+1​=argπmax​(rπ​+γPπ​vπk​​)

在元素级别上，这一步可以写为：

πk+1(s)=arg⁡max⁡π∑aπ(a∣s)(∑rp(r∣s,a)r+γ∑s′p(s′∣s,a)vπk(s′))\pi_{k+1}(s) = \arg\max_{\pi} \sum_a \pi(a|s) \left( \sum_r p(r|s,a) r + \gamma \sum_{s'} p(s'|s,a) v_{\pi_k}(s') \right) πk+1​(s)=argπmax​a∑​π(a∣s)(r∑​p(r∣s,a)r+γs′∑​p(s′∣s,a)vπk​​(s′))

实际上，这一步的最优策略是贪心策略，即在每个状态 sss 下，选择使 qπk(s,a)q_{\pi_k}(s, a)qπk​​(s,a) 最大的动作：

ak∗(s)=arg⁡max⁡aqπk(a,s)a_k^*(s) = \arg\max_a q_{\pi_k}(a, s) ak∗​(s)=argamax​qπk​​(a,s)

其中

qπk(a,s)=∑rp(r∣s,a)r+γ∑s′p(s′∣s,a)vπk(s′)q_{\pi_k}(a, s) = \sum_r p(r|s, a) r + \gamma \sum_{s'} p(s'|s, a) v_{\pi_k}(s') qπk​​(a,s)=r∑​p(r∣s,a)r+γs′∑​p(s′∣s,a)vπk​​(s′)

于是，新的策略为

πk+1(a∣s)={1,a=ak∗(s)0,a≠ak∗(s)\pi_{k+1}(a|s) = \begin{cases} 1, & a = a_k^*(s) \\ 0, & a \neq a_k^*(s) \end{cases} πk+1​(a∣s)={1,0,​a=ak∗​(s)a=ak∗​(s)​

策略迭代算法的整个过程会产生一系列的策略和值函数：

π0→PEvπ0→PIπ1→PEvπ1→PIπ2→PEvπ2→PI⋯\pi_0 \xrightarrow{\text{PE}} v_{\pi_0} \xrightarrow{\text{PI}} \pi_1 \xrightarrow{\text{PE}} v_{\pi_1} \xrightarrow{\text{PI}} \pi_2 \xrightarrow{\text{PE}} v_{\pi_2} \xrightarrow{\text{PI}} \cdots π0​PE​vπ0​​PI​π1​PE​vπ1​​PI​π2​PE​vπ2​​PI​⋯

其中，PE表示策略评估（Policy Evaluation），PI表示策略改进（Policy Improvement）。

每一次策略改进都会使得新的策略不劣于甚至优于上一次的策略。事实上，可以证明，如果

πk+1=arg⁡max⁡π(rπ+γPπvπk),\pi_{k+1} = \arg\max_{\pi} (r_\pi + \gamma P_\pi v_{\pi_k}), πk+1​=argπmax​(rπ​+γPπ​vπk​​),

那么

vπk+1≥vπkv_{\pi_{k+1}} \geq v_{\pi_k} vπk+1​​≥vπk​​

对于所有 kkk 都成立。由于状态价值是单调递增且有上界，最终必然收敛。更进一步，可以证明，策略迭代算法生成的状态价值序列 {vπk}\{v_{\pi_k}\}{vπk​​} 一定收敛到最优状态价值 v∗v^*v∗，而策略序列 {πk}\{\pi_k\}{πk​} 也会收敛到最优策略。

在实际实现中，策略评估步骤本身也可以用迭代法近似求解。对于每个状态 sss，不断迭代

vπk(j+1)(s)=∑aπk(a∣s)(∑rp(r∣s,a)r+γ∑s′p(s′∣s,a)vπk(j)(s′))v_{\pi_k}^{(j+1)}(s) = \sum_a \pi_k(a|s) \left( \sum_r p(r|s,a) r + \gamma \sum_{s'} p(s'|s,a) v_{\pi_k}^{(j)}(s') \right) vπk​(j+1)​(s)=a∑​πk​(a∣s)(r∑​p(r∣s,a)r+γs′∑​p(s′∣s,a)vπk​(j)​(s′))

直到 jjj 足够大或两次迭代的差异足够小为止。

策略改进步骤则是对每个状态 sss，计算所有动作的 qπk(s,a)q_{\pi_k}(s, a)qπk​​(s,a)，选择最大者作为新的策略。

整个算法的伪代码如下：

  1. 初始化概率模型 p(r∣s,a)p(r|s, a)p(r∣s,a) 和 p(s′∣s,a)p(s'|s, a)p(s′∣s,a)，以及初始策略 π0\pi_0π0​。
  2. 对于每一次迭代 kkk，重复以下步骤直到收敛：
    • 策略评估：对每个状态 sss，迭代计算 vπk(s)v_{\pi_k}(s)vπk​​(s)，直到收敛。
    • 策略改进：对每个状态 sss，计算所有动作的 qπk(s,a)q_{\pi_k}(s, a)qπk​​(s,a)，选择最大者作为新的策略。

通过一个简单的例子可以直观理解策略迭代的过程。假设有两个状态 s1s_1s1​ 和 s2s_2s2​，奖励设置为 rboundary=−1r_{\text{boundary}} = -1rboundary​=−1，rtarget=1r_{\text{target}} = 1rtarget​=1，折扣因子 γ=0.9\gamma = 0.9γ=0.9。初始策略可能是让 s1s_1s1​ 向左，s2s_2s2​ 保持不变。通过策略评估，可以解出

vπ0(s1)=−1+γvπ0(s1)v_{\pi_0}(s_1) = -1 + \gamma v_{\pi_0}(s_1) vπ0​​(s1​)=−1+γvπ0​​(s1​)

vπ0(s2)=0+γvπ0(s1)v_{\pi_0}(s_2) = 0 + \gamma v_{\pi_0}(s_1) vπ0​​(s2​)=0+γvπ0​​(s1​)

直接解得 vπ0(s1)=−10v_{\pi_0}(s_1) = -10vπ0​​(s1​)=−10，vπ0(s2)=−9v_{\pi_0}(s_2) = -9vπ0​​(s2​)=−9。然后进行策略改进，计算 qπ0(s,a)q_{\pi_0}(s, a)qπ0​​(s,a)，选择能带来最大回报的动作，得到新的策略。经过一次迭代后，策略就已经达到最优。

在更复杂的环境下，策略迭代同样适用。比如在一个更大的网格世界中，随着策略和状态价值的不断更新，策略会逐步趋于最优，状态价值也会逐步提升。每一次策略评估和改进，都会让智能体的行为更加合理，最终收敛到最优策略和最优状态价值。

策略迭代算法的强大之处在于它的理论保证和实际效果。它不仅能够保证收敛到最优解，而且每一步的策略改进都能带来价值提升。与值迭代算法相比，策略迭代每次都进行完整的策略评估，通常收敛速度更快，但每次迭代的计算量也更大。两者各有优劣，实际应用中可以根据问题规模和计算资源灵活选择。
