强化学习基本概念

重要概念

state

state （状态）: 智能体（agent）在某一时刻对环境的全部观测或描述。

该例子中state就指的是location，即s1,s2…,s9，也就是当前所在的格子坐标 (x, y)

state space（状态空间） :所有可能状态的集合，记作 S\mathcal{S}S。

Action

Action（动作） : 在某个状态下，智能体可以选择的操作。

该例子中Action指的是a1,a2…,a5

Action space : 所有可能动作的集合，记作 A\mathcal{A}A。和状态是依赖的。

A(si)={ak}k=15.\mathcal{A}(s_i) = \{ a_k \}_{k=1}^5.A(si​)={ak​}k=15​. A\mathcal{A}A 是 sis_isi​ 的一个函数

state transition

state transition(状态转移）指的是智能体（agent）在环境中采取某个动作（action）后，环境从一个状态（state）转变到另一个状态的过程。

forbidden area

forbidden area（禁区、不可达区域）: 指的是智能体（agent）在环境中不能进入或不能通过的区域。这些区域在环境建模时被明确标记为“禁止访问”，如果智能体尝试进入这些区域，一般会有以下2种处理方式：

  • 惩罚：可以进入，但如果进入禁区，给予较大的负奖励（penalty），甚至直接终止本次试验（episode）。

  • 直接阻止：智能体无法移动到该区域（比如是一堵墙）动作无效，位置不变。

（在该博客中大多指第 1 种情况，更加的一般化，也更加的难）

  • 如果禁区的惩罚不是“无限大”，而是一个有限的负奖励，在某些情况下智能体会选择“穿越禁区”，以获得更短的路径或更高的长期回报。例如：如果绕开禁区要多走10步，每步-1分，而穿越禁区只需-5分，智能体可能会选择“冒险”穿越禁区。
  • 如果禁区惩罚设置不合理，智能体可能陷入局部最优（比如宁愿一直在禁区里“受罚”，也不去探索更优路径）。这对奖励设计提出了更高要求。

举例说明

假设 grid-world 如下（S为起点，G为终点，X为禁区）：

  • 进入X格，奖励-10；每走一步，奖励-1；到G奖励+20。

  • 如果S到G的最短路径必须穿过X，智能体会权衡“走禁区”还是“绕远路”。

  • 如果禁区惩罚很大，智能体会尽量绕开；如果惩罚较小，智能体可能会选择“穿越禁区”以更快到达目标。

state transition probability

状态转移概率描述了在给定当前状态和动作的情况下，环境转移到下一个状态的概率，是强化学习和马尔可夫决策过程（MDP）中的一个核心概念。

数学表达式为：P(s′∣s,a)P(s' \mid s, a)P(s′∣s,a)

  • sss：当前状态（state）

  • aaa：当前动作（action）

  • s′s's′：下一个状态（next state）

  • P(s′∣s,a)P(s' \mid s, a)P(s′∣s,a)：在状态sss下采取动作aaa后，转移到状态s′s's′的概率

有时也写作 P(s,a,s′)P(s, a, s')P(s,a,s′) 或 Pss′aP_{ss'}^aPss′a​。

理解：就是“你在某个状态下做了某个动作，环境会以多大概率变成下一个状态”。

  • 在确定性环境中，这个概率要么是1（必然转移），要么是0（不可能转移）。

  • 在随机环境中，这个概率可以是0到1之间的任意值。

状态转移概率是建模环境动态的基础。在模型RL算法（如动态规划、值迭代、策略迭代）中，必须已知或能估计状态转移概率。在无模型RL算法（如Q-learning）中，状态转移概率不需要显式已知，但它仍然隐含在环境中。

policy

policy（策略）是智能体（agent）在每个状态下选择动作的“行为准则”或“决策规则”。

数学表示

随机策略（Stochastic Policy）

  • 读作：“在状态 sss 下采取动作 aaa 的概率”。

  • π(a∣s)\pi(a|s)π(a∣s) 是一个概率分布，满足 ∑aπ(a∣s)=1\sum_a \pi(a|s) = 1∑a​π(a∣s)=1。

确定性策略（Deterministic Policy）

  • 读作：“在状态 sss 下，策略 π\piπ 选择动作 aaa”。

  • 这里 π\piπ 是一个从状态空间到动作空间的映射。

举例说明

  • π(a∣s)=0.8\pi(a|s) = 0.8π(a∣s)=0.8 表示在状态 sss 下有80%概率选择动作 aaa。

  • π(s)=a2\pi(s) = a_2π(s)=a2​ 表示在状态 sss 下总是选择动作 a2a_2a2​。

    • 确定性策略：a=π(s)a = \pi(s)a=π(s)
